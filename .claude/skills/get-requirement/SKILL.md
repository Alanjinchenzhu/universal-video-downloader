# get-requirement 技能 - 获取项目需求文档

## ⚠️ 功能概述

当用户请求获取项目需求文档时触发，本技能会：
1. 扫描项目根目录及常用文档目录
2. 查找并收集需求相关文档
3. 整理并展示项目需求信息

## 扫描的文档类型

| 文档类型 | 文件模式 | 说明 |
|---------|---------|------|
| 需求文档 | `*requirement*.md`, `*需求*.md`, `REQ*.md` | 需求文档 |
| 设计文档 | `*design*.md`, `*设计*.md`, `DESIGN*.md` | 设计文档 |
| 项目说明 | `README.md`, `PROJECT.md` | 项目概述文档 |
| 技术文档 | `*doc*.md`, `*文档*.md` | 其他相关文档 |
| CLAUDE配置 | `.claude/CLAUDE.md` | 项目配置文档 |

## 扫描的目录

优先扫描以下目录：
- 项目根目录 `/`
- `/docs/`
- `/doc/`
- `/documentation/`
- `/requirements/`
- `/specs/`

## 执行步骤

### 步骤 1：搜索需求相关文档

使用 Glob 工具搜索以下文件模式：
```
**/*requirement*.md
**/*需求*.md
**/REQ*.md
**/*design*.md
**/*设计*.md
**/DESIGN*.md
**/README.md
**/PROJECT.md
**/.claude/CLAUDE.md
```

### 步骤 2：读取找到的文档

对于找到的每个文档，使用 Read 工具读取其内容。

### 步骤 3：整理并展示结果

按照以下格式整理并展示结果：

```markdown
## 📋 项目需求文档汇总

### 📁 找到的文档列表
| 序号 | 文件名 | 路径 | 类型 |
|------|--------|------|------|
| 1 | [文件名] | [路径] | [类型] |

---

### 📄 文档内容

#### [文档1名称]
[文档内容摘要或完整内容]

---

#### [文档2名称]
[文档内容摘要或完整内容]
```

### 步骤 4：分析项目需求（可选）

如果未找到明确的需求文档，根据以下信息分析并生成需求摘要：
1. 从 CLAUDE.md 读取项目信息
2. 从 README.md 读取项目概述
3. 扫描项目代码结构，推断功能模块

## 输出要求

- 使用中文展示
- 清晰列出找到的所有文档
- 对于长文档，提供摘要或关键部分
- 如果未找到需求文档，明确告知用户并提供建议
